# 投稿作成ツール — 要件定義書

## 1. プロダクト概要

### 1.1 目的
ローンチコンセプト設計ツールで作成したコンセプトシートを元に、媒体別・フェーズ別の投稿コンテンツをAIと壁打ちしながら1つずつ高品質に作成し、投稿スケジュールとタスクを管理するWebアプリケーション。

### 1.2 解決する課題
- コンセプトは決まったが、具体的な投稿が作れない
- 媒体ごとの文体調整が面倒（LINE短文 vs メール長文）
- 毎日の投稿ネタ切れ
- 何をいつ投稿するかの管理ができていない
- 投稿の質にバラつきがある（マーケティング知識が不足）

### 1.3 ユーザー
- ローンチコンセプト設計ツールのユーザー（個人事業主・フリーランス）
- 自分用 + 他の人にも渡せるレベル

### 1.4 提供価値
- コンセプトシートに基づいた一貫性のある投稿を量産
- マーケティング・コピーライティングの専門知識をAIが内蔵
- 1投稿ずつ壁打ちして仕上げるので品質が高い
- カレンダー＋タスク管理で「やるべきこと」が明確

## 2. 機能要件

### 2.1 コンセプトシート読み込み

#### 2.1.1 読み込み方法
- **HTML貼り付け（推奨・高精度）**: コンセプトアプリの📋コピーボタンでコピーしたHTMLを貼り付け
- **PDFアップロード（補助）**: テキスト抽出して読み込み（精度は落ちる）
- **テキスト直接入力**: コンセプト情報を手打ちで入力

#### 2.1.2 読み込み後の処理
- コンセプトシートからターゲット像、スタイルシート（OK/NG表現）、メインコピー、発信戦略を自動抽出
- 抽出結果をプロジェクト設定として保存
- AIのシステムプロンプトに自動反映

### 2.2 投稿カレンダー

#### 2.2.1 カレンダー自動生成
- コンセプトシートの発信スケジュールに基づき、1週間分の投稿カレンダーを自動生成
- 媒体 × フェーズの組み合わせで投稿スロットを配置
- 手動で追加・削除・並べ替え可能

#### 2.2.2 カレンダー表示
- 週間ビュー（メイン）
- 各スロットに: 媒体アイコン、テーマ、ステータスバッジ
- フェーズごとの色分け

#### 2.2.3 投稿チェーン設計
- 複数の投稿を連鎖として設計
- 例: Day1 X問題提起 → Day2 IG共感 → Day3 メール解決策 → Day4 LINE CTA
- チェーン内の投稿は線でつながって可視化
- 1つのメッセージを媒体をまたいで段階的に届ける設計

### 2.3 投稿作成（壁打ち）

#### 2.3.1 媒体選択

| 媒体 | 形式 | 文量目安 |
|------|------|---------|
| Instagram フィード | 投稿文 + ハッシュタグ | 300-500字 |
| Instagram リール | シーン構成 + テロップ + キャプション | シーン別台本 |
| X 短文 | ツイート | 140字以内 |
| X 長文 | 長文ツイート | 500-1000字 |
| YouTube | 構成 + トーク台本 | 2000-3000字 |
| スレッズ | 投稿文 | 300-500字 |
| メール | ハウスリスト配信 | ~1000字（教育+価値提供+行動喚起） |
| LINE | 配信文 | 短文クリティカル（行動喚起特化） |

#### 2.3.2 フェーズ選択（手動切替）

| フェーズ | 目的 | トーン |
|---------|------|-------|
| pre-preローンチ | 認知・問題提起 | 悩み共感・危機感・あるある |
| preローンチ | 解決策提示・期待値上げ | 成功事例・理想像・証明 |
| ローンチ期間 | セミナー誘導・クロージング | 限定性・後悔回避・CTA |

#### 2.3.3 壁打ちフロー
1. カレンダーから投稿スロットを選択（or 新規作成で媒体・フェーズを選択）
2. AIがコンセプトシート＋スタイルシートを参照して初稿を生成
3. ユーザーがフィードバック（「もっと煽って」「トーン柔らかく」「具体例入れて」）
4. AIが修正 → 納得いくまで繰り返し
5. 確定して保存

#### 2.3.4 AIの行動原則（最重要）

**マーケティング知識の内蔵:**
- プリンセスマーケティング（女性向けターゲットの場合）
- ストーリーフォーミュラ（PAS、AIDA、QUEST等）
- 感情トリガー活用（恐怖、憧れ、希少性、社会的証明、損失回避）
- 認知バイアスの活用
- コピーライティングの型（見出し、ブレット、CTA）

**スタイル準拠:**
- コンセプトシートのOK/NG表現を厳守
- 「その人らしさ」を崩さない
- ターゲットの悩み・感情トリガーを自然に織り込む
- AI臭い文章にしない（場面・行動・セリフ・感情の4要素）

**媒体別の自動調整:**
- LINE: 短文・クリティカル・行動喚起特化。余計な説明なし
- メール: 1000字程度・価値提供と教育を入れた上で行動喚起
- X短文: ワンパンチで刺す。余韻を残す
- X長文: ストーリー型・教育型。最後にCTA
- Instagram: ビジュアル前提。共感→展開→CTA
- リール: シーン割り。冒頭3秒で掴む構成
- YouTube: 構成→トーク台本。視聴維持を意識
- スレッズ: 本音トーン。Xの転用+独自の切り口

### 2.4 バリエーション生成

#### 2.4.1 概要
1つのテーマから切り口を3パターン生成する機能。

#### 2.4.2 切り口パターン例
- **共感切り口**: 「わたしもそうだった」系
- **危機感切り口**: 「このままだと...」系
- **証拠切り口**: 「受講生の声」「数字で証明」系
- **ストーリー切り口**: 「あの日こんなことがあって」系
- **逆張り切り口**: 「実は○○は必要ない」系

#### 2.4.3 使い方
- 投稿作成画面で「バリエーション生成」ボタン
- 3パターン表示 → 気に入ったものを選択 → そこから壁打ちで仕上げる
- 同じメッセージを角度を変えて何度も刷り込むのに使う

### 2.5 スワイプファイル

#### 2.5.1 概要
過去に反応が良かった投稿（自分の＋参考投稿）を保存し、AIが新しい投稿を作る時に参照する機能。

#### 2.5.2 保存する情報
- 投稿テキスト
- 媒体
- カテゴリ（共感系 / 教育系 / CTA系 / ストーリー系 等）
- 参考元（自分 / 他者 / 不明）
- メモ（なぜ反応が良かったか）

#### 2.5.3 使い方
- 投稿作成時に「スワイプファイルから参考にする」ボタン
- カテゴリ・媒体でフィルタリング
- 選択するとAIが「この投稿の構造・トーンを参考にして」と指示に含める
- 手動追加 + 作成済み投稿からワンクリック登録

### 2.6 CTAテンプレート管理

#### 2.6.1 概要
フェーズごとに使うCTA（行動喚起文）を事前登録し、投稿作成時に自動挿入。

#### 2.6.2 テンプレート構造

| フェーズ | CTA例 |
|---------|-------|
| pre-pre | 「プロフィールのリンクから無料で受け取ってね」 |
| pre | 「セミナーの詳細はこちら↓」「まずは無料セミナーに来てみて」 |
| ローンチ | 「残り○枠」「今日23:59まで」「申込はプロフィールリンクから」 |

#### 2.6.3 使い方
- 設定画面でフェーズ別に複数登録
- 投稿作成時にフェーズに応じて候補を表示
- 選択するとAIが投稿末尾に自然に組み込む
- 手動で上書き可能

### 2.7 投稿管理

#### 2.7.1 投稿一覧
- 作成済み投稿の一覧表示
- フィルター: 媒体別 / フェーズ別 / ステータス別
- コピーボタン（そのままSNSに貼れる）
- 編集・削除

#### 2.7.2 ステータス管理
- **下書き**: 作成中・壁打ち中
- **確定**: 壁打ち完了、投稿待ち
- **投稿済み**: 投稿完了

#### 2.7.3 タスク管理
- 未完了タスク表示（「あと○投稿作成が必要」）
- 期限設定（投稿予定日）
- 期限が近いものを上に表示
- 進捗バー（今週の投稿作成進捗: 8/12完了）

### 2.8 素材ライブラリ

#### 2.8.1 概要
コンセプトアプリと同様に過去の素材をアップロードしてAIが参照。

#### 2.8.2 用途
- 過去の反応が良かった投稿
- メールキャンペーンの文面
- セミナー文字起こし
- お客さんの声
- マーケティングナレッジ

#### 2.8.3 一括インポート
- 指定フォルダからテキストファイルを一括読み込み
- コンセプトアプリと同じ仕組み

## 3. 画面構成

### 3.1 画面一覧

| 画面 | パス | 説明 |
|------|------|------|
| ダッシュボード | `/` | プロジェクト一覧 |
| 新規プロジェクト | `/projects/new` | コンセプトシート読み込み+プロジェクト作成 |
| プロジェクト | `/projects/[id]` | カレンダー+投稿一覧（メイン画面） |
| 投稿作成 | `/projects/[id]/posts/new` | AIと壁打ちして投稿を作成 |
| 投稿編集 | `/projects/[id]/posts/[postId]` | 既存投稿の編集・壁打ち続行 |
| スワイプファイル | `/swipe-files` | スワイプファイル管理 |
| CTA設定 | `/projects/[id]/settings` | CTAテンプレート・プロジェクト設定 |
| 素材ライブラリ | `/library` | 素材管理 |
| 使い方ガイド | `/guide` | マニュアル |

### 3.2 API一覧

| API | メソッド | 説明 |
|-----|---------|------|
| `/api/chat` | POST | AI壁打ちチャット（ストリーミング） |
| `/api/generate-calendar` | POST | 投稿カレンダー自動生成 |
| `/api/generate-variations` | POST | バリエーション3パターン生成 |
| `/api/generate-draft` | POST | 初稿の自動生成 |
| `/api/fetch-url` | POST | URL内容取得 |
| `/api/import-materials` | GET | 素材一括インポート |

## 4. データモデル

### 4.1 Project
```typescript
{
  id: string;
  name: string;
  conceptSheet: string;        // コンセプトシートHTML
  extractedData: {             // 自動抽出データ
    target: string;            // ターゲット像
    mainCopy: string;          // メインコピー
    style: string;             // スタイルシート（OK/NG表現）
    schedule: string;          // 発信スケジュール
    channels: string[];        // 使用媒体
  };
  ctaTemplates: CTATemplate[]; // CTAテンプレート
  createdAt: string;
  updatedAt: string;
}
```

### 4.2 Post（投稿）
```typescript
{
  id: string;
  projectId: string;
  channel: 'instagram_feed' | 'instagram_reel' | 'x_short' | 'x_long' | 'youtube' | 'threads' | 'email' | 'line';
  phase: 'pre_pre' | 'pre' | 'launch';
  title: string;              // テーマ・タイトル
  content: string;            // 投稿本文
  hashtags?: string;          // ハッシュタグ（Instagram用）
  reelScript?: ReelScene[];   // リール台本（Instagram Reel用）
  status: 'draft' | 'confirmed' | 'posted';
  scheduledDate?: string;     // 投稿予定日
  chainId?: string;           // 投稿チェーンID
  chainOrder?: number;        // チェーン内の順番
  swipeFileRef?: string;      // 参考にしたスワイプファイルID
  variationOf?: string;       // バリエーション元の投稿ID
  createdAt: string;
  updatedAt: string;
}
```

### 4.3 ReelScene（リール台本のシーン）
```typescript
{
  sceneNumber: number;
  duration: string;           // "3秒"
  visual: string;             // 画面に映すもの
  telop: string;              // テロップテキスト
  narration: string;          // ナレーション
}
```

### 4.4 CalendarSlot（カレンダースロット）
```typescript
{
  id: string;
  projectId: string;
  date: string;               // "2026-05-01"
  channel: Post['channel'];
  phase: Post['phase'];
  theme: string;              // テーマ
  postId?: string;            // 作成済みの場合リンク
  chainId?: string;
}
```

### 4.5 SwipeFile（スワイプファイル）
```typescript
{
  id: string;
  content: string;            // 投稿テキスト
  channel: string;            // 媒体
  category: 'empathy' | 'education' | 'cta' | 'story' | 'proof' | 'other';
  source: 'self' | 'reference';
  memo: string;               // なぜ反応が良かったか
  createdAt: string;
}
```

### 4.6 CTATemplate
```typescript
{
  id: string;
  phase: Post['phase'];
  text: string;               // CTA文
  label: string;              // 管理用ラベル
}
```

### 4.7 ChatMessage
```typescript
{
  id: string;
  postId: string;             // どの投稿の壁打ちか
  role: 'user' | 'assistant';
  content: string;
  timestamp: string;
}
```

## 5. 非機能要件

### 5.1 技術スタック
| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 16（App Router） |
| UI | React 19 + TypeScript |
| スタイリング | Tailwind CSS v4 |
| AI | Anthropic Claude API（Sonnet 4） |
| ストレージ | ブラウザ localStorage |
| アイコン | Lucide React |
| Markdown | react-markdown |

### 5.2 デザイン
- コンセプトアプリと統一感のあるダークテーマ
- 背景: #0D0D0D
- テキスト: #E8E0D5
- アクセントカラー: #C8A96E（ゴールド）
- 各媒体に固有カラー（Instagram=#E1306C, X=#1DA1F2, YouTube=#FF0000 等）

### 5.3 ポート
- 開発: localhost:3901（コンセプトアプリの3900と共存）

## 6. AIシステムプロンプトに内蔵するナレッジ

### 6.1 マーケティングフレームワーク
- PAS（Problem-Agitation-Solution）
- AIDA（Attention-Interest-Desire-Action）
- QUEST（Qualify-Understand-Educate-Stimulate-Transition）
- ストーリーフォーミュラ（ヒーローズジャーニー簡易版）

### 6.2 女性向けマーケティング
- プリンセスマーケティングの原則
- 共感ファーストの文章構成
- 「教える」ではなく「一緒に」のスタンス

### 6.3 認知バイアス活用
- 希少性（残り○名）
- 社会的証明（○名が参加）
- 損失回避（今やらないと...）
- バンドワゴン効果
- アンカリング（通常価格 vs 今回価格）

### 6.4 媒体別ベストプラクティス
- Instagram: リールの冒頭3秒、フィードの改行ルール、ハッシュタグ戦略
- X: フック→展開→CTA、引用RT誘発、長文スレッドの構成
- YouTube: サムネ→冒頭→本編→CTAの構成、視聴維持テクニック
- LINE: 短文・絵文字の使い方・タップ誘導
- メール: 件名→冒頭→本文→CTA・開封率を上げるテクニック

### 6.5 過去素材からの学習
- 素材ライブラリの投稿パターンをAIが参照
- スワイプファイルの構造・トーンを学習
